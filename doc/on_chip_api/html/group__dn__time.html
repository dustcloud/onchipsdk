Description

Get the current time and receive time pin notifications.

Note
Refer to the On-Chip Firmware Development Guide for a high level description of the time pin service.
The On-Chip Firmware Development Guide contains code snippets and sample applications, some of which use this device.

This service allows your application to retrieve the current time, and receive a notification through the local interface whenever the time pin is asserted on your device. The time pin is a single, dedicated, pin on your device.

"Asserting" the time pin is done by some external device, which sets the pin low, then high. The notification is generated on the low-to-high transition. While the pin is low, the device consumes more energy in order to return a highly accurate timestamp when the pin transitions to high. Your application should therefore keep the pin high by default, and lower it only for a small amount of time, but not less than the tstrobe value listed in the LTC5800-IPM datasheet.

Note
Refer to the LTC5800-IPM datasheet for timing and consumption characteristics associated with the time pin.

By default, the time pin service is enabled, i.e. your application does not need to enable this service.

When the time pin is asserted, your application receives a notification of type dn_api_loc_notif_time_t through the local interface.

If you are using the Local Interface Module, you need to enable these notifications by calling dnm_loc_registerTimeNotifCallback() and specifying a callback function. This callback function is called each time the time pin is asserted.

The notification indicates the Absolute Slot Number (ASN) of the time slot when the time pin was triggered, and the offset within that timeslot.
