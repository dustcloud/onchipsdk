Description

Inter-Integrated Circuit (I2C) digital bus device.

Note
Use the Device API to control this device. This page details the parameters to pass to the functions of that API.
The On-Chip Firmware Development Guide contains code snippets and sample applications, some of which use this device.

The following table indicates which Device API functions apply to this device. Calling one of the unsupported functions on this device returns DN_ERR_NOT_SUPPORTED. Only the supported functions are detailed below.

dn_open dn_ioctl dn_read dn_write dn_close
YES YES NO NO YES

Open the device

You need to call dn_open() to open this device before using any of the other functions. To open this device, use the dn_open() function with the following parameters:

Parameter Name Parameter Value
device DN_I2C_DEV_ID
args Points to a pre-populated dn_i2c_open_args_t structure.
argLen Size of dn_i2c_open_args_t.

Change the I2C frequency

You can change the frequency used by the I2C device without having to close and reopen it by issuing an dn_ioctl() function with the following parameters:

Parameter Name Parameter Value
device DN_I2C_DEV_ID
request DN_IOCTL_I2C_SETFREQ
args Points to a pre-populated dn_i2c_ioctl_frequency_t structure.
argLen Size of dn_i2c_ioctl_frequency_t.

Communicate with an I2C slave

An I2C transaction can consist of mix of transmission and reception over the I2C bus. For that reason, the transfer operation is done using the Device API's dn_ioctl() function rather than dn_read() or dn_write().

To initiate a transfer, use the dn_ioctl() function with the following parameters:

Parameter Name Parameter Value
device DN_I2C_DEV_ID
request DN_IOCTL_I2C_TRANSFER
args Points to a pre-populated dn_ioctl_i2c_transfer_t structure.
argLen Size of dn_ioctl_i2c_transfer_t.

This triggers a transfer of the data specified in the args parameter. See below for examples on how to transmit to and receive from an I2C slave.

dn_ioctl() returns one of the following values:

  • DN_ERR_NONE if the transfer completes successfully.
  • DN_ERR_BUSY if the I2C bus is already in use.
  • DN_ERR_NO_RESOURCES if the slave answers with a NACK.
  • DN_ERR_TIMEOUT if the transfer times out.
  • DN_ERR_ERROR if the transfer could not be completed.

Call this function again for subsequent transfers.

Example 1: Sending data to an I2C slave

We assume your application wants to send bytes [0x01,0x02,0x03] to an I2C slave at address 0x05. It therefore needs to populate the dn_ioctl_i2c_transfer_t structure as follows:

  • Set dn_ioctl_i2c_transfer_t::slaveAddress to 0x05, the address of the I2C slave you wish to send the data to over the I2C bus.
  • dn_ioctl_i2c_transfer_t::writeBuf points to a buffer of length 3 bytes. The 3 bytes of this buffer contain the payload to transmit: [0x01,0x02,0x03].
  • Set dn_ioctl_i2c_transfer_t::readBuf to NULL.
  • Set dn_ioctl_i2c_transfer_t::writeLen to 3, the length of the dn_ioctl_i2c_transfer_t::writeBuf buffer.
  • Set dn_ioctl_i2c_transfer_t::readLen to 0.
  • Set dn_ioctl_i2c_transfer_t::timeout to the timeout, in milli-seconds, appropriate for your application.

Example 2: Reading data from an I2C slave

We assume your application wants to read 3 bytes from an I2C slave at address 0x05. It therefore needs to populate the dn_ioctl_i2c_transfer_t structure as follows:

  • Set dn_ioctl_i2c_transfer_t::slaveAddress to 0x05, the address of the I2C slave you wish to read data from over the I2C bus.
  • Set dn_ioctl_i2c_transfer_t::writeBuf to NULL.
  • dn_ioctl_i2c_transfer_t::readBuf points to a buffer of length 3 bytes. Clear those 3 bytes, they will contain the payload received from the slave device.
  • Set dn_ioctl_i2c_transfer_t::writeLen to 0.
  • Set dn_ioctl_i2c_transfer_t::readLen to 3, the length of the dn_ioctl_i2c_transfer_t::readBuf buffer.
  • Set dn_ioctl_i2c_transfer_t::timeout to the timeout, in milli-seconds, appropriate for your application.

Example 3: Sending and receiving in one I2C transaction

You can use this device to write data to and read data from an I2C slave in a single I2C transaction. The device will first send the data specified in the dn_ioctl_i2c_transfer_t::writeBuf buffer to the I2C slave, then read data from the I2C slave and store the received data in the dn_ioctl_i2c_transfer_t::readBuf buffer.

We assume your application wants to send bytes [0x01,0x02,0x03], and read 3 bytes from an I2C slave at address 0x05, in a single I2C transaction. It therefore needs to populate the dn_ioctl_i2c_transfer_t structure as follows:

  • Set dn_ioctl_i2c_transfer_t::slaveAddress to 0x05, the address of the I2C slave you wish to send the data to over the I2C bus.
  • dn_ioctl_i2c_transfer_t::writeBuf points to a buffer of length 3 bytes. The 3 bytes of this buffer contain the payload to transmit: [0x01,0x02,0x03].
  • dn_ioctl_i2c_transfer_t::readBuf points to a buffer of length 3 bytes. Clear those 3 bytes, they will contain the payload received from the slave device.
  • Set dn_ioctl_i2c_transfer_t::writeLen to 3, the length of the dn_ioctl_i2c_transfer_t::writeBuf buffer.
  • Set dn_ioctl_i2c_transfer_t::readLen to 3, the length of the dn_ioctl_i2c_transfer_t::readBuf buffer.
  • Set dn_ioctl_i2c_transfer_t::timeout to the timeout, in milli-seconds, appropriate for your application.

Close the device

Close the device by calling the dn_close() function.

Warning
This device should only be closed after your application is fully done using it. Reopening this device after having closed it will fail and return a DN_ERR_STATE error.
