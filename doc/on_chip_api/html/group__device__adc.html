Description

Analog-to-Digital Converter devices.

Note
Use the Device API to control these devices. This page details the parameters to pass to the functions of that API.
The On-Chip Firmware Development Guide contains code snippets and sample applications, some of which use these devices.

The following table indicates which Device API functions apply to these devices. Calling one of the unsupported functions on these devices returns DN_ERR_NOT_SUPPORTED. Only the supported functions are detailed below.

dn_open dn_ioctl dn_read dn_write dn_close
YES YES YES NO YES

Overview

The Analog-to-Digital Converter (ADC) devices enable your application to acquire readings from several internal and external analog sources, amplify them, apply offset correction, and digitize them.

Use this module to use the ADC on the following devices:

  • DN_ADC_AI_0_DEV_ID
  • DN_ADC_AI_1_DEV_ID
  • DN_ADC_AI_2_DEV_ID
  • DN_ADC_AI_3_DEV_ID
  • DN_BATT_DEV_ID
  • DN_GND_DEV_ID
  • DN_VREF_DEV_ID
Note
While sampling the battery voltage and internal temperature sensor does use the same ADC hardware module, refer to the Battery and Temperature documentation, respectively.
Warning
The ADC is designed to sample an input voltage in the 0-1.8V range. Never apply a voltage outside that range to an ADC input channels. Doing so will damage the part.

Open the device

To open an ADC device, call the dn_open() function with the following parameters:

Parameter Name Parameter Value
device The ADC channel of interest, taken from dn_device_t.
args Points to a pre-populated dn_adc_drv_open_args_t struct.
argLen Size of dn_adc_drv_open_args_t.

Read from the device

Precondition
Before using the function, make sure to open the device using dn_open().

To read the voltage of the ADC channel, call the dn_read() function with the following parameters:

Parameter Name Parameter Value
device The ADC channel of interest, taken from dn_device_t.
buf Points to a variable of type INT16U.
bufSize Size of INT16U.

After this function returns, the INT16U variable contains the voltage read from the ADC channel, in tenth of a milli-volt. That is, a value of 12300 corresponds to a voltage of 1.23V.

Note
Refer to the On-Chip Firmware Development Guide for a description of the VGA and RDAC.

Close the device

Close the device by calling the dn_close() function. This will release the device and allow a (possibly different) task to open and use it.

Using the RDAC and VGA

Refer to the On-Chip Firmware Development Guide for documentation about the RDAC and VGA.

You have two options for controling the setting of the RDAC and VGA:

  • when opening the device, through the dn_adc_drv_open_args_t::rdacOffset, dn_adc_drv_open_args_t::vgaGain and dn_adc_drv_open_args_t::fBypassVga elements passed in the dn_open() function.
  • when the device is already open, through the dn_ioctl() function. This is explained in the paragraphs below.

To control the RDAC once the device is already open, use the dn_ioctl() function with the following parameters:

Parameter Name Parameter Value
device The ADC channel of interest, taken from dn_device_t.
request DN_IOCTL_RDAC_OFFSET_CFG
args Points to a pre-populated dn_adc_drv_ioctl_rdac_t structure.
argLen Size of dn_adc_drv_ioctl_rdac_t.

To control the VGA once the device is already open, use the dn_ioctl() function with the following parameters:

Parameter Name Parameter Value
device The ADC channel of interest, taken from dn_device_t.
request DN_IOCTL_VGA_GAIN_CFG
args Points to a pre-populated dn_adc_drv_ioctl_vga_t structure.
argLen Size of dn_adc_drv_ioctl_vga_t.
