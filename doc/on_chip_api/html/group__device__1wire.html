Description

1-Wire digital bus device.

Note
Use the Device API to control this device. This page details the parameters to pass to the functions of that API.
The On-Chip Firmware Development Guide contains code snippets and sample applications, some of which use this device.

The following table indicates which Device API functions apply to this device. Calling one of the unsupported functions on this device returns DN_ERR_NOT_SUPPORTED. Only the supported functions are detailed below.

dn_open dn_ioctl dn_read dn_write dn_close
YES YES YES YES YES

Overview

You can use the 1-Wire protocol with the following devices:

  • DN_1WIRE_UARTC0_DEV_ID (not recommended since prevents use of CLI)
  • DN_1WIRE_UARTC1_DEV_ID
  • DN_1WIRE_UART_DEV_ID
Note
Refer to the On-Chip Firmware Development Guide for the pins corresponding to those different devices.

Internally, the 1-Wire devices are multiplexed with other devices. The table below lists the conflicts between devices. Using the device in the "option 1" column prevents your from using the device in the "option 2" column at the same time.

option 1 option 2
DN_1WIRE_UARTC0_DEV_ID DN_CLI_DEV_ID on DN_CLI_PORT_C0 (see CLI)
DN_1WIRE_UARTC1_DEV_ID DN_CLI_DEV_ID on DN_CLI_PORT_C1 (see CLI)
DN_1WIRE_UART_DEV_ID DN_UART_DEV_ID (see UART)

Open the device

You need to call dn_open() to open this device before using any of the other functions. To open this device, use the dn_open() function with the following parameters:

Parameter Name Parameter Value
device One of the 1-Wire devices from the Overview section.
args Points to a pre-populated variable of type dn_ow_open_args_t.
argLen Size of dn_ow_open_args_t.

The dn_ow_open_args_t::maxTransactionLength sets the maximum number of bytes you will ever send or receive in one transaction over 1-Wire. This value is used in the 1-Wire drivers to reserve memory for internal buffering. We therefore strongly recommend to match the value to the actual maximum transaction length (e.g. 8 bytes).

Detect the presence of at least one slave

Precondition
Before using the function, make sure to open the device using dn_open().

As per the 1-Wire specification, a 1-Wire transaction starts by the master sending a reset pulse. Each 1-Wire slave device answers with a presence pulse. When a presence pulse is received, the 1-Wire master knows that at least one 1-Wire slave is present on the 1-Wire bus.

We call "detection" the combination of the reset pulse and the presence pulse, if any. You can perform this detection by using the dn_ioctl() function with the following parameters:

Parameter Name Parameter Value
device One of the 1-Wire devices from the Overview section.
request DN_IOCTL_1WIRE_DETECT
args Points to a variable of type dn_ow_ioctl_detect_t.
argLen Size of dn_ow_ioctl_detect_t.

dn_ioctl() returns DN_ERR_NONE if the detection was performed successfully. In that case, dn_ow_ioctl_detect_t::slavePresent is 0x01 if at least one slave is present, or 0x00 if no slaves are present.

Detection can fail if no pull-up resistor is present on the 1-Wire line. In this case, dn_ioctl() returns DN_ERR_HW.

Read bytes from a 1-Wire device

Precondition
Before using the function, make sure to open the device using dn_open().

To read from a 1-Wire device, prepare a buffer to receive the bytes in. Then, call the dn_read() function with the following parameters:

Parameter Name Parameter Value
device One of the 1-Wire devices from the Overview section.
buf Points to the buffer you allocated.
bufSize The number of bytes of the buffer you allocated.

Write bytes to a 1-Wire device

Precondition
Before using the function, make sure to open the device using dn_open().

To write to a 1-Wire device, prepare a buffer with the bytes you wish to send over the 1-Wire bus. Then, call the dn_write() function with the following parameters:

Parameter Name Parameter Value
device One of the 1-Wire devices from the Overview section.
buf Points to the buffer you allocated.
len The number of bytes of the buffer you allocated.

Read a single bit

Precondition
Before using the function, make sure to open the device using dn_open().

It is something useful to be able to read a single bit from a 1-Wire bus, for example when doing a search operation. The more generic dn_read() function will read bytes, not bits. To read a single bit, use the dn_ioctl() function with the following parameters:

Parameter Name Parameter Value
device One of the 1-Wire devices from the Overview section.
request DN_IOCTL_1WIRE_READBIT
args Points to a variable of type dn_ow_ioctl_readbit_t.
argLen Size of dn_ow_ioctl_readbit_t.

After calling this function, the dn_ow_ioctl_readbit_t.readData field contains the bit read from the 1-Wire bus, either 0x00 or 0x01.

Write a single bit

Precondition
Before using the function, make sure to open the device using dn_open().

To write a single bit to the 1-Wire bus, declare a variable of type dn_ow_ioctl_writebit_t and set the bit you want to send (either 0x00 or 0x01) in the dn_ow_ioctl_writebit_t::writeData field.

Then, call the dn_ioctl() function with the following parameters:

Parameter Name Parameter Value
device One of the 1-Wire devices from the Overview section.
request DN_IOCTL_1WIRE_WRITEBIT
args Points to the variable of type dn_ow_ioctl_writebit_t.
argLen Size of dn_ow_ioctl_writebit_t.

Close the device

Close the device by calling the dn_close() function.

Warning
This device should only be closed after your application is fully done using it. Reopening this device after having closed it will fail and return a DN_ERR_STATE error.
