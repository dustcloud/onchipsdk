Description

Universal Asynchronous Receiver/Transmitter (UART) serial port device.

Note
Use the Device API to control this device. This page details the parameters to pass to the functions of that API.
The On-Chip Firmware Development Guide contains code snippets and sample applications, some of which use this device.
The UART uses HDLC encoding.

The following table indicates which Device API functions apply to this device. Calling one of the unsupported functions on this device returns DN_ERR_NOT_SUPPORTED. Only the supported functions are detailed below.

dn_open dn_ioctl dn_read dn_write dn_close
YES NO NO NO YES

Open the device

You need to call dn_open() to open this device before using any of the other functions. To open this device, use the dn_open() function with the following parameters:

Parameter Name Parameter Value
device DN_UART_DEV_ID
args Points to a variable of type dn_uart_open_args_t.
argLen Size of dn_uart_open_args_t.

Receiving over UART

The SmartMesh stack sends received UART frames over an asynchronous channel associated with message type DN_MSG_TYPE_UART_NOTIF.

To use UART, your application needs to create a task which listens for these types of messages, and handles them:

  • in the task's initialization code (i.e. before the infinite loop):
    • create an asynchronous notification channel (see Channel).
    • use the dn_registerChannel() function to associate message type DN_MSG_TYPE_UART_NOTIF to that channel.
    • open the UART device, as described in Open the device.
  • in the task's body (i.e. inside the infinite loop):
    • Call dn_readAsyncMsg() to wait for a received UART frame from the asynchronous channel.
    • Handle the UART frame as appropriate for your application.
    • Loop to pend on the semaphore once again, ready to receive the next UART frame.

Transmitting over UART

The SmartMesh stack automatically creates a synchronous channel for your application to send UART frames, and associates that channel to message type DN_MSG_TYPE_UART_TX_CTRL.

To send a frame over UART, use dn_sendSyncMsgByType() with message type set to DN_MSG_TYPE_UART_TX_CTRL.

Close the device

Close the UART device by calling the dn_close() function. This will release the device and allow a (possibly different) task to open and use it.
